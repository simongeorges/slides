C'est quoi une migration ?

  • Une source
  • Une destination
  • Un process

Quelques cas d'utilisation de Migrate

  • RSS (Stop waiting for Feeds)
  • Import massif de données
  • Migration d'un site à un autre

Des modules importants

  • Migrate Tools (pour aller plus vite)
  • Migrate Plus (pour aller plus loin)

Migrate Plus

  • Configuration Entity "Migration"
  • Classe "SourcePluginExtension"
  • Plugin source "Url"
  • Plusieurs plugins de process
    • Entity Lookup & Entity Generate
    • Merge
    • SkipOnValue

Mais aussi

  • Des types de plugins
    • Data fetcher (File, Http)
    • Data parser (Simple Xml, Xml, Json, Soap)
    • Authentication (OAuth, Basic, Digest)

Quelques exemples de codes

(Avouez que vous êtes là pour ça)

Une source simple : CSV


source:
  plugin: csv
  delimiter: ,
  path: 'public://data.csv'
  header_row_count: 1
  keys:
    - Name

Une source complexe : le RSS


source:
  plugin: url
  data_fetcher_plugin: http
  urls: 'https://example.com/feed.rss'
  data_parser_plugin: simple_xml
  item_selector: /rss/channel/item
  fields:
    -
      name: guid
      label: GUID
      selector: guid
    -
      name: image
      label: Image
      selector: 'itunes:image[''href'']'

Une destination simple


destination:
  plugin: entity:node

une destination complexe


destination:
  plugin: entity:taxonomy_term
  default_bundle: my_vocabulary
  translations: true

Un process simple


process:
  title: Title
  type:
    plugin: default_value
    default_value: page
  empty: { }

Un process medium


process:
  'body/value': body
  'body/summary': summary

Un process complexe


process:
  field_my_taxonomy:
    -
      plugin: explode
      source: My taxonomy
      delimiter: ","
    -
      plugin: skip_on_empty
      method: process
    -
      plugin: entity_lookup
      entity_type: taxonomy_term
      value_key: 'name'

Des plugins utiles

Format Date


process:
  created:
    plugin: format_date
    from_format: 'D, d M Y H:i:s O'
    to_format: 'U'
    source: pub_date

Skip On Empty


process:
  # This field does not exist, just here to filter data.
  skip_filter:
    -
      plugin: skip_on_empty
      method: row
      source: type
    -
      plugin: skip_on_value
      method: row
      value: 'formation'
      not_equals: true
      source: type

Static Map


process:
  'body/format':
    plugin: static_map
    source: description_format
    map:
      'text/plain': plain_text
      'text/html': full_html
      'text/x-web-markdown': markdown
      'text/x-rst': rst

Paramètres dynamiques


$migration = Migration::load('audits');
if (migration) {
  $source = $migration->get('source');
  $source['urls'] = $my_urls;
  $migration->set('source', $source);
  $migration->save();
}

Lancer la migration


$migration = \Drupal::service('plugin.manager.migration')
  ->createInstance($my_migration_id);
$executable = new MigrateExecutable($migration,
  new MigrateMessage());
$executable->import();

Résolution de problèmes

Mes modifications ne sont pas prises en compte


$migrations = ['my_migration_1','my_migration_2',];
foreach ($migrations as $migration) {
  Drupal::configFactory()
    ->getEditable('migrate_plus.migration.' . $migration)
    ->delete();
}

Je veux modifier les données de la source


function hook_migrate_prepare_row();
function hook_migrate_MIGRATION_ID_prepare_row();

Il ne se passe rien quand je lance la migration

Avez-vous activé "Migrate Source CSV" ?

Je veux importer aussi les éléments qui ont changé


source:
  plugin: SOME_PLUGIN
  # 'drush migrate-import --update' fait ça
  # pour TOUTES les lignes
  track_changes: true

Mon plugin est appelé plusieurs fois


/**
 * @MigrateProcessPlugin(
 *   id = "my_plugin",
 *   handle_multiples = TRUE
 * )
 */

Mon plugin de process doit se lancer au début de l'import


implements ImportAwareInterface
  -> preImport() / postImport()
implements RollbackAwareInterface
  -> preRollback() / postRollback()
